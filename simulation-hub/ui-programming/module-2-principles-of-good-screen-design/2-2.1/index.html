Theory Refresher

Information Architecture (IA)

The practice of deciding how to arrange the parts of something to be understandable. It centers on structure, navigation, and labeling.

Organization

Grouping related content cards logically (e.g., placing Pay Fees and Fee Receipts together under finance categories) to mirror real student behaviors.

Labeling

Creating clear and simple category names so that users instantly recognize what they contain without reading helper details.

Navigation

The physical path users traverse to discover content. A predictable layout minimizes clicks and path backtracking.

Search & Clear Grouping

Predictable folder groups facilitate quick visual searches, reducing the student's cognitive load and preventing user frustration.

Observations

Group pages and menus logically to help users build a correct mental model. Run an answer check to see live analysis.

Conclusion

A clear application structure improves usability by making pages predictable, organized, and easy to access.
