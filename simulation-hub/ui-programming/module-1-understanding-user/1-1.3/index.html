Core Principles & Metaphors

• Mental models: The internal understanding user forms of a system’s operation. Design layout must align with standard expectations.
• Real-world metaphors: Familiar real-world icons (Trash for delete, Folder for files, Gear for settings) reduce system learning curves.
• Task analysis: Decomposing a user goal into discrete, minimal, logical sequential steps. Prevents visual clutter.

Standards & User Involvement

• ISO Standards:
  - ISO 9241: Ergonomic displays & software usability.
  - ISO 14915: Multimedia interface ergonomics.
  - ISO 13407: Human-centered design lifecycle processes.
  - ISO/CD 20282: Usability of everyday products.
• User Involvement:
  - Direct Methods: Interviews, field observations, testing, prototyping.
  - Indirect Methods: Online surveys, log file audits, support ticket analysis.

Consultant’s Observation: Complete challenges to generate UX insights. When interfaces match users’ mental models, use familiar metaphors, and follow clear task flows, users can complete tasks faster with less confusion.

Consultant’s Conclusion

Human-centered design improves usability by matching the interface with user expectations, real-world understanding, task flow, and design standards.
