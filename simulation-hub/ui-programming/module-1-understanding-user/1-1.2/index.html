📚 Human Heuristics Notebook

Why People Struggle with Computers

1. Non-obvious layout design: Hidden features that fail standard discovery.
2. Fine distinctions: Using tiny differences in icons or labels that are hard to spot.
3. Problem-solving strategy mismatch: Systems demanding complex technical concepts from normal users.
4. Design inconsistency: Changing structures across pages, forcing continuous relearning.

Cognitive Limits & Memory Load

• Short-term memory limits: Working memory holds only 7±2 chunks of information at once (Miller's Law).
• Mental models: Users construct expectations based on prior experience. Breaches trigger confusion.
• Psychological responses: Confusion scales to annoyance, escalates to frustration, peaks as panic/stress, and results in boredom/abandonment.
• Physical/behavioural responses: Users physically abandon the task or use only a tiny subset of features.

Analyst's Observation: Complete challenges to generate HCD insights. When interfaces overload memory, use inconsistent patterns, or hide important actions, users become confused, frustrated, and may abandon the system.

Analyst's Conclusion

Good design respects human limitations by reducing memory load, using consistent patterns, showing clear feedback, and organizing information in a way users can easily understand.
