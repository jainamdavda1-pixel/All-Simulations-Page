THEORYLocal vs Global Scope

Core Concepts:

Variables reside in isolated namespaces based on where they are initialized:

  • Local Scope: Variables created inside a function are local to it. They are created on function invocation and garbage collected immediately when the function terminates.
  • Global Scope: Variables initialized at the module root level. They are accessible by any function in the module for reading, but modifying them requires explicit keywords.
  • LEGB Lookup Rule: When resolving a variable name, Python searches namespaces in strict order: **L**ocal → **E**nclosing → **G**lobal → **B**uilt-in.

THEORYOverriding Scope: global & nonlocal

Core Concepts:

By default, trying to assign a value inside a function to a variable declared outside creates a *new local* variable rather than modifying the outer one. Keywords change this behavior:

  • global: Declares that a variable inside the function refers to the module-level global namespace.
  • nonlocal: Declares that a variable refers to the nearest enclosing scope (excluding global). Vital for nested functions holding state.

THEORYNested Functions & Closures

Core Concepts:

A nested function is a function defined inside another outer function:

  • Inner Namespace Access: Inner functions can read variables from their outer enclosing function (via Enclosing scope).
  • Closures: A closure is a record storing a function together with an environment. Even after the outer function finishes executing, the inner function continues to "remember" and access variables from the outer scope's namespace.

Closure Constructor Pattern

def make_multiplier(factor): def multiply(x): return x * factor # factor is captured from enclosing scope return multiply double = make_multiplier(2) double(5) # returns 10

THEORYDecorators & Closure Wrapper Scopes

Core Concepts:

A decorator is a function that takes another function as an argument, extends its behavior without modifying it, and returns a new function (closure wrapper):

  • Wrapper Function: The nested inner function that receives arguments, executes the wrapped function, and optionally transforms findings.
  • Argument Delegation (*args, **kwargs): Allows the decorator wrapper to accept any arbitrary positional or keyword arguments and forward them seamlessly.
  • Function Return: The outer decorator function must return the inner wrapper function reference (without calling it).

THEORYVariable Shadowing & LEGB Resolving

Core Concepts:

Variable shadowing occurs when a variable declared within a local scope has the same name as one in an outer scope:

  • Shadowing Action: The local variable "shadows" (hides) the outer variable. Inside the function, the name resolves to the local value, leaving the outer variable untouched.
  • Debugging Shadowing: Unintentional shadowing (like overriding built-in names like list, id, or str) is a common source of bugs in Python applications.
