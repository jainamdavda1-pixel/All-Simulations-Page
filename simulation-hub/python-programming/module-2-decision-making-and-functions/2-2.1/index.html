THEORYSimple if & if-else Branches

Core Concepts:

Execution branching allows a program to choose different paths based on conditions:

  • Conditional Execution: Code inside an if block evaluates only when its Boolean condition is True.
  • Indentation Syntax: Python does not use braces to define scopes. It strictly relies on indentation (4 spaces).

Comparison Reference

Operator Meaning Example → Result

THEORYelif Chains and Mutually Exclusive Paths

Core Concepts:

Chaining conditions with elif handles multi-way branches:

  • Order Matters: Python checks conditions top-to-bottom and exits the entire chain on the first matches.

elif Chain Structure

score = 72 if score >= 90: grade = "A" elif score >= 80: grade = "B" elif score >= 70: grade = "C" # matches first

THEORYNested Conditions & Logical Operators

Core Concepts:

Compound conditions combine Boolean paths:

  • and: Requires both operands to evaluate to True.
  • or: Requires at least one operand to be True.
  • not: Negates the condition check.

THEORYTernary Expressions & the pass Statement

Core Concepts:

Inline expressions compress structural paths:

  • Ternary Shorthand: x = val_if_true if condition else val_if_false.
  • pass statement: Syntactical placeholder code when empty blocks are required.

THEORYTruthiness & Structural Pattern Matching

Core Concepts:

Implicit values dictate logical branching paths:

  • Falsy variables: 0, 0.0, empty containers ([], "", None, and False).
  • Truthy variables: Everything else.
