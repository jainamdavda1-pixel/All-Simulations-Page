THEORYThe while Loop Counter

Core Concepts:

A while loop repeatedly executes a block as long as its target condition remains True:

  • Pre-Evaluation: The condition is evaluated *before* entering the loop block. If it is initially False, the block never executes.
  • Infinite Loop Risk: If the condition variables are not updated within the loop body to eventually yield False, the loop will run forever, locking up interpreter memory.

  • range(start, stop, step): Generates a sequence of integers. Crucially, the stop boundary is **exclusive**.

THEORYLoop Modifiers: break & continue

Core Concepts:

Altering iteration scopes dynamically:

  • break: Exits the enclosing loop block immediately.
  • continue: Skips current block statement logic and proceeds to the next iteration.

THEORYParallel Streams & Nested Unpacking

Core Concepts:

Unlocking advanced sequence traversal patterns in Python:

  • enumerate(iterable): Yields index-element pairs: (index, item).
  • zip(*iterables): Pairs corresponding items from multiple sequences. It terminates at the length of the shortest iterable.
  • Zipped Tuple Unpacking: Combining them as enumerate(zip(A, B)) yields nested structures like (idx, (itemA, itemB)), requiring parentheses for correct syntax unpacking in the loop header.
