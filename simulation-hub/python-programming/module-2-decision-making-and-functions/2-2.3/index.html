THEORYDefining & Invoking Functions

Core Concepts:

Functions are named, modular blocks of code that perform specific tasks:

  • def Keyword: Declares functions: def function_name(params):.
  • return Keyword: Exits scope and sends value back. Defaults to None.

Function Anatomy

def greet(name): message = "Hello, " + name return message

THEORYRequired vs Keyword vs Default Arguments

Core Concepts:

Binding parameters dynamically during calls:

  • Required variables: Must be supplied by positional inputs.
  • Default values: Fallbacks. Must be declared *after* required parameters.

THEORYVariable-Length Arguments: *args & **kwargs

Core Concepts:

Arguments collectors pack arbitrary bounds:

  • *args: Packs extra positional arguments into a **tuple**.
  • **kwargs: Packs extra keyword arguments into a **dictionary**.

THEORYInput Guards, Type Hints & PEP 484

Core Concepts:

Assertions and typing structures protect variables:

  • Type hints: Optional metadata annotations used by static analysis tools. Ignored at runtime.
  • Guards: Raise errors programmatically to stop math calculations on invalid objects.
