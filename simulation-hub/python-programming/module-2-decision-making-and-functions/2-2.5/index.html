THEORYAnonymous Expressions: lambda

Core Concepts:

A lambda function is a lightweight, one-line anonymous function defined without a name:

  • Syntax Constraints: Declared using lambda arguments: expression. No explicit def or return keywords are used. The evaluated expression is implicitly returned.
  • Single Expression Limit: Lambdas can contain *only one* expression. They cannot contain statements (such as assignments = or loops for).

THEORYFunctional Mapping & Filtering

Core Concepts:

Python provides functional programmers with mapping and filtering utilities to transform arrays:

  • map(function, iterable): Applies the target mapping function to every single item in the input sequence.
  • filter(function, iterable): Evaluates each sequence item against a Boolean function, retaining only elements that yield True.
  • Lazy Evaluation: Both map and filter return lazy iterator generators. To manifest their concrete results into memory, you must wrap them with list().

THEORYRecursion & Context Stack Frames

Core Concepts:

Recursion is a programming technique where a function calls itself:

  • Base Case (The Stop Guard): A conditional branch that returns a value without making further recursive calls, preventing infinite recursion.
  • Recursive Case (Reduction): The path where the function calls itself with a reduced subset of parameters, moving closer to the base case.
  • Stack Limits: Each recursive call creates a new context frame on the execution stack. To prevent stack overflows, Python limits default recursion depth to 1000 frames.

THEORYMemory References: Shallow vs Deep Copy

Core Concepts:

How objects duplicate inside memory affects reference isolation:

  • Assignment (=): Copies the memory address reference only. Both variables point to the exact same object. Modifying one modifies the other.
  • Shallow Copy (copy.copy): Creates a new outer container, but nested elements (like sublists) remain shared references. Modifying a nested list modifies both objects.
  • Deep Copy (copy.deepcopy): Fully duplicates all structures recursively, creating a completely independent copy in memory. Modifying any nested element leaves the original unchanged.

THEORYDecorators & Call Interception

Core Concepts:

A decorator is a meta-programming utility used to extend the behavior of a function without modifying its source code:

  • Wrapper Function: Decorators are functions that take another function as an argument, wrap it with extra logic (e.g. logging execution speed or validating parameters), and return the wrapper function.
  • Sugar Syntax: Placing @decorator_name directly above a function declaration is syntactic sugar for func = decorator_name(func).
