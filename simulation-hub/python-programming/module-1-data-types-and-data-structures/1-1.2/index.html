THEORYDivision Mechanics & Modulo Arithmetic

Core Concepts:

Python distinguishes between different mathematical divisions and modulo behaviors to maintain mathematical consistency:

  • True Division (/): Always returns a floating-point number, even if the result divides perfectly (e.g., 10 / 2 = 5.0).
  • Floor Division (//): Divides the operands and rounds the result downward toward negative infinity (floor value). For positive numbers, this acts as truncation (e.g., 17 // 5 = 3). However, with negative numbers, it rounds down (e.g., -17 // 5 = -4).
  • Modulo (%): Returns the remainder of division. It satisfies the relation a = (a // b) * b + (a % b). When divisors are negative, the result matches the sign of the divisor (e.g., 5 % -3 = -1). Modulo on negatives (e.g., -17 % 5 = 3) is highly useful in cyclic scheduling and modular structures.
  • Exponentiation (**): Computes powers (e.g., 2 ** 3 = 8). Floating-point powers compute fractional roots (e.g., 9 ** 0.5 = 3.0) and widen the resulting type.

THEORYLogical Operators & Short-Circuit Evaluation

Core Concepts:

Python's logical operators exhibit distinct behaviors optimized for performance and expression evaluation:

  • Truthy vs. Falsy Values: In Python, empty collections (e.g. [], {}), the number zero (0, 0.0), empty strings (""), and the singleton None evaluate to False in a logical context. All other values evaluate to True.
  • Short-Circuiting Rules:
    • or evaluates expressions left-to-right. The moment it encounters a truthy operand, it stops evaluating and returns that operand's value immediately.
    • and evaluates expressions left-to-right. The moment it encounters a falsy operand, it short-circuits and returns that operand's value. If all operands are truthy, it returns the final operand.
  • Non-Boolean Returns: Unlike many languages where logical operators return strict boolean values (True or False), Python's and and or operators return the value of the actual operand that resolved the logical operation.

THEORYRounding Traps & Built-in vs. Module Math

Core Concepts:

Python provides math operations at two distinct layers: native global built-in functions, and the specialized, optimized standard library math module:

  • Built-in Rounding Limits: Python's global round(number, ndigits) function utilizes **"Banker's Rounding"** (round-to-even) under IEEE 754. For instance, round(2.5) and round(1.5) both evaluate to 2. This avoids statistical bias by rounding halfway cases to the nearest even integer.
  • Floor vs. Ceil Boundaries:
    • math.floor(x): Rounds down to the nearest integer toward negative infinity.
    • math.ceil(x): Rounds up to the nearest integer toward positive infinity.
    • For negative values, these shift directions: math.floor(-5.67) = -6, whereas math.ceil(-5.67) = -5.
  • Function Performance: Standard utilities like abs(), min(), and max() are implemented in optimized C inside builtins. Special transcendental operations (trigonometry, factorials, logs) are packed into the math module, operating on raw float conversions.

THEORYBitwise Manipulation & Floating-Point Limits

Core Concepts:

Bitwise operations manipulate numbers at the raw binary bit level. Floating-point arithmetic, on the other hand, deals with constraints imposed by physical memory boundaries:

  • Bitwise AND (&): Compares bits of two numbers; returns 1 only if both bits are 1. Used to check if specific flags/permissions are active.
  • Bitwise OR (|): Returns 1 if at least one corresponding bit is 1. Used to combine permission sets or toggle flags ON.
  • Bitwise XOR (^): Returns 1 if corresponding bits are different (one is 1, other is 0). Perfect for toggling state/status flags.
  • Bit Shifts (<< / >>):
    • Left shift (x << y) shifts bits to the left, filling vacancies with 0s. This is equivalent to multiplying x by 2y.
    • Right shift (x >> y) shifts bits to the right. This is equivalent to integer floor division of x by 2y.
  • IEEE 754 Floating-Point Precision: Python represents floats using 64-bit double-precision binary values. Numbers like 0.1 and 0.2 have infinite repeating representations in binary, similar to 1/3 in base-10. This leads to subtle rounding offsets (e.g. 0.1 + 0.2 = 0.30000000000000004). To bypass this, we use round() for display or the standard library decimal.Decimal for financial calculations.

CONCEPTFloat Precision Trap

print(0.1 + 0.2) 0.30000000000000004 ← NOT 0.3! # Fix: use round() for display print(round(0.1 + 0.2, 2)) # 0.3 # Or: use the decimal module for exact arithmetic from decimal import Decimal print(Decimal('0.1') + Decimal('0.2')) # 0.3

THEORYOperator Precedence & Walrus Binding Mechanics

Core Concepts:

Precedence rules control the order in which expressions are resolved in Python. Bypassing these rules requires parenthetical overrides:

  • Precedence hierarchy (High to Low):
    1. Parentheses: ( )
    2. Exponentiation: **
    3. Multiplication/Division/Modulus: *, /, //, %
    4. Addition/Subtraction: +, -
    5. Bitwise Shifts: <<, >>
    6. Comparisons: <, >, ==, !=, etc.
    7. Logical Not: not
    8. Logical And: and
    9. Logical Or: or
  • Chained Comparisons: Python allows shorthand mathematical syntax like 1 < x < 10. This is evaluated as (1 < x) and (x < 10). It executes short-circuiting: if the first comparison fails, the second expression is never checked.
  • The Walrus Operator (:=): Introduced in PEP 572, this "assignment expression" operator binds a value to a variable and returns that value inside a single expression. Because it has very low precedence, you must wrap it in parentheses when used in operations (e.g. if (n := len(x)) > 5:).

CONCEPTWalrus Operator & Chained Comparisons

# Walrus := assigns AND returns the value if (n := len("hello")) > 3: print(f"Length {n} is big enough") # Chained comparisons — very Pythonic x = 7 print(1 < x < 10) # True — Python chains these! print(1 < x < 5) # False
